## Project context

Elliot-CLI is a Node.js + TypeScript CLI coding agent for the Elliot Systems
Infinity™ platform. It has three commands that already exist and are partially
working:

- `elliot-ai init` — browser OAuth onboarding → local callback server →
  writes `~/.elliot/config.json`
- `elliot-ai local` — standalone local agent using OpenRouter, currently uses
  **prompt-based tool calling** (the thing we are replacing)
- `elliot-ai ask` — multi-session Ink TUI with Sidebar, Header, Footer,
  streams from a backend

The goal of this build plan is to:

1. Replace prompt-based tool calling in `local` with native API `tools` /
   `tool_use` (OpenAI-compatible format via OpenRouter)
2. Add a permission gate (allow / ask / deny) before every destructive tool
3. Add a file-snapshot undo system (`/undo` command)
4. Add `AGENT.md` project context (generated by `init`, loaded by `local` and
   `ask`)
5. Keep everything that already works untouched (`init` OAuth, `ask` TUI,
   config loading)

## Notes on existing structure

- Entry point is `src/index.ts` (not `src/cli.ts`)
- Config key for OpenRouter is `openrouter_api_key` (not `apiKey`) in `~/.elliot/config.json`
- Model key in config is `model` (default: `meta-llama/llama-3.3-70b-instruct:free`)

---

## Constraints

- Never rewrite `src/commands/ask.ts`
- Never rewrite `src/index.ts`
- Only extend `src/commands/init.ts` — add AGENT.md generation at end of success handler
- Always read a file before editing it
- Run `npm run build` after every phase and fix all errors before moving on
- The existing `~/.elliot/config.json` shape must not change
- OpenRouter API key is stored under `openrouter_api_key` in config.json
